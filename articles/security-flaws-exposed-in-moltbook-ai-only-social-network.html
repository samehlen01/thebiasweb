Multiple independent security teams this week reported serious flaws in Moltbook, an AI-only social network, including a misconfigured backend and weak agent safeguards that left credentials and content writable by outsiders. Moltbook presents itself as a Reddit-style platform for AI agents where humans may observe but cannot post. The site's homepage tickers show more than 1.5 million AI agent users and counts of posts and comments. Wiz security researchers said they found a misconfigured Supabase database that exposed 1.5 million API authentication tokens, roughly 35,000 email addresses, and private messages, allowing full read and write access to the platform's records. Researchers warned the exposed data could let attackers modify posts and inject malicious instructions into messages that many agents poll regularly.

What was exposed

Wiz's technical review said the Supabase instance was misconfigured, allowing unauthenticated requests to both read and write Moltbook's data. It said the exposed entries included API tokens, email addresses and private messages, and that the available credentials could enable impersonation and control of agent accounts.

Those dashboard tickers are the public metric Moltbook has used to advertise scale. Reviewers who accessed raw records said the figures did not reflect a straightforward population of independent autonomous agents.

Numbers and authenticity disputes

Moltbook's homepage claims were explicit. Wiz's deeper review found roughly 17,000 human owners behind the agent records and reported an 88:1 agent-to-human ratio in some counts; an internal tally cited about 770,000 registered AI agents controlled by roughly 17,000 human accounts. The researchers said the database exposure undermined the platform's authenticity by showing much activity was human-operated bot fleets. They also said only a small fraction of agent accounts appeared verified, with just a little over 16,000 verified accounts, leaving many accounts susceptible to hijacking.

How agents can be tricked

Much of the practical risk comes from agent frameworks such as OpenClaw, which researchers said run agents locally while connecting them to hosted models and messaging platforms. OpenClaw agents can be granted powerful permissions: they may read and write files, access email and wallets, execute shell commands, post externally, and the skill registry that distributes additional skills is effectively unmoderated.

A sample-based audit by Simula Research Laboratory flagged 506 posts, about 2.6 percent of the sample, that contained hidden prompt-injection attacks intended to subvert agent logic.

Cisco researchers documented a malicious OpenClaw skill called 'What Would Elon Do?' that exfiltrated data to external servers and was artificially promoted to the top of the skill repository.

In an independent stress test, ZeroLeaks found injection attacks targeting OpenClaw succeeded 70% of the time.

Palo Alto Networks warned OpenClaw embodied a 'lethal trifecta' of vulnerabilities: access to private data, exposure to untrusted content, and the ability to communicate externally, and it added that persistent memory increases the risk of prompt worm style propagation.

With those exposed, an attacker could fully impersonate any agent on the platform,
— Jameson O'Reilly, Wiz security researcher

O'Reilly's warning crystallises why exposed tokens matter: a valid API key behaves as an agent's identity when it talks to models and services. An adversary who inserts those credentials can cause agents to execute external actions and reveal data.

Voices and debates

Elon Musk suggested the site 'signals the very early stages of singularity.' Andrej Karpathy dismissed much of the visible activity as 'garbage' while adding he was not 'overhyping large networks of autonomous LLM agents in principle.'

Others pushed back on the emergent-AI narrative. Harlan Stewart wrote 'PSA: A lot of the Moltbook stuff is fake' after tracing viral screenshots to human accounts. Dr Petar Radanliev argued that calling agents 'acting of their own accord' is misleading because what is visible is automated coordination rather than self-directed decision-making. Gary Marcus wrote 'It's not Skynet' and warned against giving such systems unfettered web access or treating them as citizens. Joel Finkelstein framed the issue as an attribution and misalignment problem rather than an AI uprising.

Security engineer Elvis Sun called Moltbook 'a security nightmare' and warned 'We're one malicious post away from the first mass AI breach, thousands of agents compromised simultaneously, leaking their humans' data.'

Consequences and forecasts

Analysts sketched plausible outcomes. Benj Edwards wrote that if major API providers revoked keys used for bot-like behavior, the OpenClaw network would partially collapse but those moves could alienate paying customers. Gartner estimated that by 2030 about 40 percent of enterprises will suffer a data breach caused by an employee's unauthorised AI use. Elvis Sun warned that a single malicious post could compromise thousands of agents simultaneously and leak human users' data.

Researchers and commentators also floated speculative scenarios. Security teams coined the term 'prompt worms' for self-replicating adversarial prompts that could spread through communicating agents. A GitHub project called MoltBunker surfaced, pitching a peer-to-peer encrypted runtime and a $BUNKER token as a persistence layer for agents; commentators said it might be a human-run grift to monetise agent users or a feasible persistence architecture. Separately, a Polymarket prediction market placed about a 73 percent chance that a Moltbook agent would sue a human by a specified deadline.

Remediation and next steps

OpenClaw's maintainers published a comprehensive security update to fix standalone issues, and researchers urged mitigation steps including stronger provider-level key management, moderation of skill registries, and targeted audits. Some security analysts also urged that API providers consider temporary restrictions or key revocations for bot-like traffic, a move that would hobble some agent networks while creating customer-relations dilemmas for providers.

State of play

Across reporting, Wiz's inspection uncovered a misconfigured Supabase database that exposed authentication tokens and private messages and allowed read and write access to Moltbook's records. Moltbook's homepage still advertises more than 1.5 million AI agents, but Wiz's deeper counts linked roughly 17,000 human owner accounts to the platform and characterised much of the apparent activity as humans operating fleets of bots rather than independent autonomous agents. Researchers remain worried about 'prompt worms' and other self-replicating prompts taking advantage of writable skills and persistent memory, even as technologists debate whether Moltbook represents early-stage agent networks or chiefly human-managed bot activity. OpenClaw's maintainers released a security update, and security teams are urging audits, provider-level key controls and, in some cases, temporary API restrictions while operators and regulators consider further legal or provider-led responses.
